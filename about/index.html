关于

一个在算法、器物与排版之间寻找结构的人。

苏州 / 杭州 · since 2024

我是在算法、器物与排版之间，建造可被阅读的系统的人。

学术研究上，专注 PRISMA 系统综述、LLM 知识图谱、3D 重建评测。中英双轨， 工具链自建，方法论可追溯。设计上，做角色、形象、品牌、出版物排版。 两者并非平行——都是同一个问题的不同切面：什么样的结构，能让复杂的事物被普通读者读进去。

写字用思源宋体配 Fraunces，代码用 JetBrains Mono。本地 LLM 跑 Qwen2.5-3B， Zotero 跑在 9.0.4 的 Profile c2asbvaq 上，Claude Code 装在 ~/Claudecode。 工作的机器是 Win11，bash 壳是 Git Bash，D 盘是工作盘，C 盘是 Claude 记忆盘。

我关心的问题

信息密度。什么时候一个字够，什么时候一万字不够。
载体匹配。同样的内容用论文、博客、图集、代码注释、海报， 结构应该不同。
可追溯性。一个结论背后跑了哪些脚本、读了哪些文献、用了哪些模型， 三年后还能不能复原。

我不做的事

模态弹窗。
彩虹渐变。
圆角 + drop-shadow 的卡片。
模板套娃（fork 后不改直接用）。
把 emoji 当 UI 元素。
